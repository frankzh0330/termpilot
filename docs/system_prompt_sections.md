# System Prompt Sections

[English](system_prompt_sections.md) | [简体中文](system_prompt_sections.zh-CN.md)

This document explains the structure built by `context.py` in `build_system_prompt()`.

## Overview

The system prompt is assembled from a mix of static sections and dynamic sections. The exact section count is less important than the design rule:

- stable core instructions stay as constants
- environment-dependent instructions are generated by helper functions

## Static Core

The static core includes guidance for:

- role and safety boundaries
- system behavior and rendering assumptions
- how to perform software engineering tasks
- how to use tools
- action safety
- tone and output efficiency

These sections define the model’s baseline operating rules.

## Dynamic Sections

Dynamic sections are generated from runtime state, including:

- session-specific guidance based on enabled tools
- memory guidance and memory index content
- environment info such as cwd, platform, shell, date, and git status
- language preference, when set
- MCP server instructions, when available
- reminders about summarizing important tool-result details

## Important Helpers in `context.py`

- `get_system_context()`
- `get_git_status()`
- `get_session_guidance_section()`
- `get_language_section()`
- `get_mcp_instructions_section()`
- `load_memory_prompt()`
- `build_system_prompt()`

## Memory Integration

The memory subsystem is not just a blob of past notes. The prompt includes:

- instructions about what kinds of memory exist
- when memory should or should not be saved
- how memory files are structured
- the current `MEMORY.md` index content when present

## MCP Integration

When connected MCP servers expose instructions, those instructions are appended as their own prompt section so the model can follow server-specific usage constraints.

## Design Goal

The prompt builder is intentionally compositional:

- each section is easier to edit in isolation
- runtime-dependent behavior stays explicit
- the overall prompt can evolve without turning into one giant template string
